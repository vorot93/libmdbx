629  * automatically (de)initialization, releasing reader lock table slots
630  * and so on.
631  *
632  * If MDBX builded as a DLL this is done out-of-the-box by DllEntry() function,
633  * which called automatically by Windows core with passing corresponding reason
634  * argument.
635  *
636  * Otherwise, if MDBX was builded not as a DLL, some black magic
637  * may be required depending of Windows version:
638  *
639  * - Modern Windows versions, including Windows Vista and later, provides
640  * support for "TLS Directory" (e.g .CRT$XL[A-Z] sections in executable
641  * or dll file). In this case, MDBX capable of doing all automatically,
642  * therefore you DON'T NEED to call mdbx_module_handler()
643  * so the MDBX_MANUAL_MODULE_HANDLER defined as 0.
644  *
645  * - Obsolete versions of Windows, prior to Windows Vista, REQUIRES calling
646  * mdbx_module_handler() manually from corresponding DllMain() or WinMain()
647  * of your DLL or application,
648  * so the MDBX_MANUAL_MODULE_HANDLER defined as 1.
649  *
650  * Therefore, building MDBX as a DLL is recommended for all version of Windows.
651  * So, if you doubt, just build MDBX as the separate DLL and don't care about
652  * the MDBX_MANUAL_MODULE_HANDLER. */
